Os navegadores aplicam estilos padrão a muitos elementos. No caso de elementos de entrada, 
sua capacidade de estilizá-los com CSS pode parecer bastante restrita. Portanto, você pode querer 
appearance: noneocultar aspectos do elemento padrão e criar o seu próprio. Por exemplo, isso 
ocultaria as caixas de seleção padrão para uma entrada de caixa de seleção, permitindo que você 
use indicadores personalizados, como um visto verde e um X vermelho, para mostrar o estado. Para
uma entrada de pesquisa, os navegadores baseados em WebKit exibirão um ícone de pesquisa padrão 
e um botão de cancelar. Ocultá-los permite que você crie seus próprios indicadores que aparecerão 
em todos os navegadores.

O WebKit é um mecanismo de software que ajuda navegadores a exibir sites. Navegadores como o Safari 
usam o WebKit para garantir que as páginas da web tenham a aparência e o funcionamento corretos. 
Esta appearance: nonepropriedade CSS oferece controle total sobre o estilo, mas também apresenta 
alguns aspectos a serem observados. Os componentes interativos padrão dos elementos de entrada 
incluem recursos como indicadores de foco e erro, que você precisa garantir que não sejam perdidos.

Criar um estilo multiplataforma consistente é um ótimo motivo para usar essa propriedade. 
Você também pode usá-la para garantir que os pontos de toque em um dispositivo móvel sejam grandes 
o suficiente ou que as cores de uma caixa de seleção tenham contraste suficiente.
